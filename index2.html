Introduction

What is React.js?

  • Developed by Facebook
  • React is a view layer library, not a framework like Backbone, Angular etc.
  • You can't use React to build a fully-functional web app.

Why was React developed?

  • Complexity of two-way data binding
  • Bad UX from using "cascading updates" of DOM tree
  • A lot of data on a page changing over time
  • Complexity of Facebook's UI architecture
  • Shift from MVC mentality.

Why React?

Declarative

  • Predictable code
  • Easy to understand what a component will render

No complex two-way data flow

  • Uses simple one-way reactive data flow
  • Easier to understand than two-way binding
  • Uses less code

React is fast!

  • Real DOM is slow
  • JavaScript is fast
  • Using virtual DOM React is faster

Server-side rendering

  • React.renderToString() returns pure HTML

React dev tools

  • React Chrome extension makes debugging so much easier
  • This tool will help us track changes in our React application

Trying Out React

  • Locally Coding Create React App (recommended)
  • Online code on CodePen (recommended)

React & ReactDOM

For simple start we've two libraries: React and ReactDOM

Import React, React-Dom

React

  • The React object is what's going to allow us to create objects that'll eventually become DOM nodes.
  • React automatically re-renders UI as our application state changes

Application State

State is the piece of information that keep track of the element present in your web application.

React

  • React library contains a myriad of functions/classes
  • React is simple
  • Main 2 functions/classes are React.createElement() and React.Component

React.createElement()

  • createElement() method creates an HTML elements

React.Component

  • Component in React is essentially a configurable, and reusable piece of UI.
  • They can be as simple as a function you write that returns a React "Element".
  • Or a class you define that extends the React.Component.

ReactDOM

  • React on its own is designed to be just a library for building User Interfaces.
  • Various platforms (ie. web, mobile applications, desktop applications, etc).
  • To build React apps for the web we need ReactDOM
  • Has render() method

ReactDOM.render()

  • This method is where we plug our React application into an HTML document.
  • 2 parameters: React Element & HTML DOM node where where React Element is to be rendered

ReactDOM.render()


		React.createElement(HTMLElement, prop, text);
		const btn = React.createElement('button', null, 'SkyZone');
		console.log(JSON.stringify(btn));

		ReactDOM.render(btn, document.querySelector('#root'));

JS vs ReactDOM.render()


		JS
		const btn = document.createElement('button');
		btn.innerText = 'SkyZone';
		document.querySelector('#root').appendChild(btn);

		ReactDOM
		const btn = React.createElement('button', null, 'SkyZone');
		ReactDOM.render(btn, document.querySelector('#root'));

Problem with JSX?

JSX is not supported by browsers.

How do we solve it?

Solution: Babel

Babel is a tool that is mainly used to convert ES6 code into a backwards compatible version of JavaScript in current and older browsers or environments.

  • Babel is a JavaScript compiler
  • Transform syntax
  • Polyfill features that are missing in your target environment
  • Source code transformations
  • And more!

Babel Sample


	  // Babel Input: ES2015(ES6) arrow function
	  [1, 2, 3].map((n) => n + 1);

	  // Babel Output: ES5 equivalent
	  [1, 2, 3].map(function(n) {
		return n + 1;
	  });
